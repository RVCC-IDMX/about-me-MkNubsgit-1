How Animated SVG Works

For this blog topic I picked animated SVG, mainly because I just made a logo for this site and figured it'd be a great excuse to make it move.

SVG is a vector image format defined in XML. Unlike a JPEG or PNG (where the image is locked to a grid of pixels), SVG describes shapes using vector representations. That has two main consequences. The image scales to any size without becoming blurry, and you can target individual shapes inside the SVG with CSS or JavaScript. SVG has existed in some capacity since the late 90s but became more used once browsers got faster at rendering and hi-DPI screens made vector graphics worth utilizing. The SVG Standard was developed by the World Wide Web Consortium. There were 6 competing proposals for vector graphics languages, which were eventually beaten out by SVG. SVG was initially not supported by Internet Explorer, limiting support. After 2011, however, all major browsers offered SVG support natively. There are also two mobile profiles of SVG, SVGT and SVGB, which were intended for platforms with less capabilities. Wikipedia on SVG

Being able to target individual shapes is what makes SVG animation possible. If your SVG is embedded inline in your HTML, every shape inside is a functional DOM element. You can give it a class, hover over it, and animate any property the browser knows how to interpolate.

There are two primary ways to animate an SVG: CSS animations (the same @keyframes you'd use for any HTML element) and SMIL animations (the older animate tag built into SVG itself). A majority of modern projects use CSS because it integrates better with the rest of the site's stylesheets. SMIL still works but has been on uncertain support ground for years.

Demo: Drawing the monogram in

This demo animates my ME logo as if it's being drawn by a pen. The technique uses two SVG path properties: stroke-dasharray, which turns the path into a dashed line, and stroke-dashoffset, which shifts where those dashes start. By setting the dash length equal to the path's total length and animating the offset from "fully hidden" to "fully visible", the line appears to draw and undraw itself. As for the logo itself, I just made it using figma earlier in the process of completing this web-dev final. W3Schools (i used it to teach myself this!)

When to use this:

Animated SVG is incredibly useful for UI elements. It's significantly lighter than an mp4 file and accessible in ways raster images aren't, since screen readers can read structured SVG content. Every major browser has supported SVG for over a decade, so there's no reason to avoid it on compatibility grounds. The downside is complexity. A long SVG animation can be a lot of markup to maintain compared to using a video file.

For something like the animated monogram on this site (right up there!), animated SVG is the right tool, especially as CSS is already styling the rest of the page.
